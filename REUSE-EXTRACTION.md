# 可复用底层逻辑 · 提取分析

做完整站双语 + 探测器模拟器后，回看这个项目里有哪些**可复用的底层逻辑**，
按「值不值得沉淀成全局资产」诚实分级。原则：不为凑数造 skill——只有真能跨项目
复用、且自成一体的，才建议提升；其余标清楚是「贴片段」还是「只是套路」。

分级：🟢 值得提升为全局 skill／独立库 · 🟡 复制即用的片段 · ⚪ 只是思路/模式

---

## 🟢 A. 整站双语引擎（最强候选，已基本成形）

**是什么**：一个零依赖的 classic script `app/public/i18n.js`（约 130 行）+ 内容仓
`content.js` + 声明式 `data-i18n` 属性，让**同一套 DOM 运行时切语言**，且
「不能 import ES module 的静态页」和「打包后的 SPA」共用同一个引擎。配套
`TRANSLATION-GUIDE.md` 定义翻译标准与「新增内容怎么接」的流程。

**为什么可复用**：任何 vanilla / Vite 多页站要做真·双语，都会撞上同一堵墙——
静态页用不了模块系统、动态字符串和数据字段也要翻。这套的解法（window 全局引擎 +
`{zh,en}` 内容对 + `data-i18n`/`-html`/`-attr` + `localStorage` 记忆 + 自动绑定的
切换按钮 + `t(key,{param})` 插值）是通用的，换个项目改改 key 命名就能用。

**可移植性**：`i18n.js` 已是自成一体的独立文件，零外部依赖，直接拷走即可。
翻译标准和接入流程已写进 `TRANSLATION-GUIDE.md`。

**配套的验收不变量**（同样可复用）：*「EN 模式下，任何带 `data-i18n*` 的元素、
以及被翻译的属性（alt / data-cap 等），其文本都不得含 CJK 字符。」* 一条
`querySelectorAll('[data-i18n]').filter(含中文)` 就能机检整站翻译是否有漏。
这把「翻译完整性」从肉眼抽查变成了可自动化的断言——比人工扫更靠谱。

**验收里踩到的通用坑**：后台标签页 `requestAnimationFrame` 会被节流到 0 帧，
导致「播放/动画类功能」在无头或非前台标签里假阴性。解法：测试时若
`document.visibilityState !== 'visible'`，临时把 rAF 打补丁成 `setTimeout(cb,16)`
再驱动。这是所有「用 rAF 做动画、又要在后台标签自测」的项目都会遇到的。

**verdict**：建议提升为全局 skill（暂名 `bilingual-static-site`）——但**创建全局
skill = 持久化自我修改，需 CEO 一句授权**（见下「待批」）。skill 内容已可直接从
本项目三件套（`i18n.js` + `TRANSLATION-GUIDE.md` + 上面的验收不变量）打包。

## 🟡 B. 天球路径平滑工具（纯函数，可整段拷）

**是什么**：`app/src/main.js:387–470` 的四个纯函数——
`catmullRom`（样条插值）、`unwrapRaPath`（解 RA 的 0/360° 回卷不连续）、
`segmentTurnDeg`（转角）、`smoothProbePath`（按转角自适应细分，只在高曲率处加点）。

**为什么可复用**：任何「球面上的折线要画成丝滑曲线」的场景都能用——赤经赤纬轨迹、
经纬度航迹、地图连线。它专门处理了两个通用难点：① 经度环绕的不连续（RA/经度过
0/360 会诈胡），② 只在弯的地方加密、直的地方不浪费点（自适应细分）。且穿过所有
原始采样点、不引入 NaN。

**可移植性**：四个纯函数、零依赖，整段拷即可。唯一要改的是输入点的字段名
（本项目是 `{ra, dec}`）。

**verdict**：复制即用的片段。太专（要有球面路径数据），单独不够格立 skill；
若以后多个天文/地图项目复用，再收进一个 `spherical-geometry` 小工具库。

## 🟡 C. 沿路径播放模拟器（动画模式，半耦合）

**是什么**：`main.js` 的 `ribbonOf` / `drawProbeRibbon(probe, fraction)` /
`probeDateAtFraction` / `startProbePlay`——把一条**烘焙好的路径**放进「分数空间」
（0→1），在 N 秒内扫过全程，一路渐进重绘 + 领头标记 + 实时日期标签，可暂停/复位。

**为什么可复用**：「点一下，沿一条既定轨迹从头走到尾」是很通用的交互——时间轴回放、
路线动画、进度演示都是这个骨架。关键设计是**把时间/进度归一到 [0,1] 分数**，渲染
和「当前值」都只依赖 fraction，于是播放、拖动滑杆、复位共用一套逻辑。

**可移植性**：骨架通用，但重绘那步耦合了渲染目标（这里是 Aladin overlay）。移植时
换掉 `drawProbeRibbon` 里的画法即可。

**verdict**：模式 + 片段。记住「归一到分数空间」这个核心，渲染层各项目自替。

## ⚪ D.「记录是弹簧、其实是直线」解释器（内容模式）

**是什么**：一张卡片 + 内联 SVG，讲清「探测器其实近直线飞，照片记录呈弹簧状 =
地球公转视差投影到天球」。

**为什么算模式**：这是一类**观测赝象解释**的样板——凡是「看到的形状 ≠ 物体真实
运动」（视差、光行差、投影畸变）都能套这个「先给现象、再拆成两层运动、配一张最简
示意图」的讲法。代码本身不通用，通用的是叙事结构。

**verdict**：只是思路。收进知识库当科普写法样板即可，不入代码资产。

## ⚪ E. JSON 字段双语化脚本（一次性迁移助手）

`scratchpad/bilingualize_anchors.js`：把数据文件里的字段幂等地包成 `{zh,en}`
（已是对象就跳过），一次写多个运行时副本。**verdict**：一次性迁移套路，
需要时现写十行即可，不必沉淀。

## 小结与「待批」

**真正够格提升的只有 A（整站双语引擎）**——它自成一体、跨项目复用面广、且已带
可自动化的验收不变量。B/C 是拷得走的片段，D/E 是套路。诚实说：其余几个单独立
skill 会是过度工程（违反 Simplicity-First），标清楚复用方式比硬造 skill 更有用。

**待 CEO 一句授权**：把 A 打包成全局 skill `bilingual-static-site`（含 `i18n.js`
模板 + 翻译标准 + 「EN 无 CJK 残留」验收断言 + rAF 后台节流坑）。创建全局 skill =
持久化自我修改，按治理规矩不自作主张，等你醒了点个头我就用 `skill-creator` 落地。
